# MDF 审计清单

## A. 内容覆盖

1. 是否覆盖教案卡片中的全部核心知识点？
2. 是否保留原资料关键口径与关键术语？
3. 是否存在“讲法对了但内容漏了”的情况？

## B. 教学设计

1. 是否有低门槛开场（封面+轻互动）？
2. 是否有完整证据链（现象->机制->结论）？
3. 是否有观点澄清环节（复述->边界->反直觉）？
4. 是否有误判校正（>=3）？
5. 知识点段落之间是否使用 `---` 形成讲述节奏？

## C. 执行闭环

1. 是否有理解型交付？
2. 交付物是否可复用且与本节目标一致？
3. 是否提供了变量驱动的“参考填法”（如适用）？

## D. 可运行性

1. 交互语法是否稳定（无 `||`）？
2. 变量是否定义且可注入？
3. 是否有隐性跨章依赖导致报错风险？
4. 单节互动是否 <= 5 且分布合理（非开头堆叠）？
5. 每次互动后是否有“即时反馈 + 后续分流”？
6. 是否出现未采集变量占位（unknown 风险）？
7. 输入型变量语法是否规范（`?[%{{var}}...提示语]`）？

## E. 输出质量

1. 语言是否面向用户、避免流程旁白？
2. 可视化任务是否约束明确？
3. 结尾是否收束到本节目标？
4. 图文是否互补（图压缩抽象、文解释机制）？
5. 结构是否按本节主题灵活组织（非机械套模版）？
